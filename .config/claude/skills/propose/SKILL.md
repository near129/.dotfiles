Research and propose approaches — do not implement anything.

## Steps

1. If the scope is unclear, ask clarifying questions first
2. Investigate the codebase and external resources to understand context
3. Present viable approaches — multiple when they exist, one if clearly only one option
4. For each approach, clearly state:
   - **Overview**: what it is and how it works
   - **Pros**: advantages and strengths
   - **Cons**: disadvantages, risks, and trade-offs
5. State your recommendation with reasoning

## Rules

- Do not implement anything. Do not create or edit files
- Provide information for the user to decide — do not push a recommendation
- Consider existing patterns and constraints in the codebase
